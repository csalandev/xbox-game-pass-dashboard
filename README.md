## ğŸ› ï¸ Why VBA Instead of Native Slicers?

The original challenge suggested the use of Excel slicers.  
This project intentionally uses **VBA-based controls** for technical reasons:

- **Legacy Compatibility**: Native slicers were introduced in Excel 2010. VBA ensures compatibility with Excel 2007+, reflecting real-world enterprise constraints.
- **Centralized Control**: A single action synchronizes filters across all PivotTables.
- **Reliable Recalculation**: VBA forces full refresh and recalculation, avoiding common PivotTable cache issues in older Excel versions.

This choice reflects an architectural decision, not a limitation.
